Dataclass and Attribute Events

  • - Execute when certain conditions are met
  • - You do not call them directly
  • - Most are available for both dataclass and attribute
  • - In all cases, the keyword this refers to the entity
  • - Attribute level events receive a parameter that is the name of the attribute
  • - Code in an event can cause another event to execute but not the same event.
    This avoids infinite loops.

Event Types


    On Init (dataclass and attribute) - common
  • - Executes when a new entity comes into existance
  • - Runs AFTER any auto-sequence or auto-generate properties are applied
  • - Runs BEFORE the entity is delivered to the creating code

    On Load (dataclass) - uncommon
  • - Executes when an existing entity is loaded from the datastore
  • - Runs BEFORE the entity is delivered to the accessing code

    On Load (attribute) - uncommon
  • - Executes the first time an attribute value is accessed or assigned
  • - Runs for both new AND existing entities
  • - Runs BEFORE the attribute's value is delivered to the accessing code

Event Types (continued)


    On Set (attribute only) - modestly common
  • - Executes when a value is assigned to an entity's attribute
  • - Runs AFTER the value has already been assigned

    On Validate (dataclass and attribute) - very common
  • - Executes when an entity is being validated or saved
  • - Runs BEFORE the On Save event if an entity is being saved
  • - Can reject the validation by returning an object
  • - Used to check attributes values and return what is wrong if need be

    On Save (dataclass and attribute) - very common
  • - Executes when an entity is being saved
  • - Runs AFTER the On Validate event
  • - Can reject the validation by returning an object
  • - Used to complete any 'busy work' needed by the application

Event Types (continued)


    On Remove (dataclass and attribute) - modestly common
  • - Executes when an entity is being removed
  • - Runs BEFORE the entity is deleted
  • - Can reject the deletion by returning an object

    On Restricting Query (dataclass only) - very common
  • - Executes when code attempts to access entities from the dataclass
  • - Returns an entity collection of the dataclass
  • - Restricts which entities can be accessed from the dataclass
  • - Used in conjunction with security and dataclass inheritance
  • - Will be explained and demonstrated in Day 3 training
