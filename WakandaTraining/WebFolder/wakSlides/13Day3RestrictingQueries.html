Restricting Queries

  • Controls which entities can be accessed from a dataclass
  • Applies to all mechanisms that access entities
  • Can be implemented in one of two ways
    • Restricting Query String
    • On Restricting Query Event

Restricting Query String

  • Property of the dataclass
  • Same syntax as a Wakanda query string
  • Most appropriate use is in derived dataclasses
  • May use a few Wakanda maintained values
    • :$userID - ID of the authenticated user
    • :$userName - Login name of the authenticated user
  • Can also use query string javascript syntax!

On Restricting Query Event

  • Allows code to restrict entity access
  • Much more flexible than a Restricting Query String
  • Goal is to return an entity collection
