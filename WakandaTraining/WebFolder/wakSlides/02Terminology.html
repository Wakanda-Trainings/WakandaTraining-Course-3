Wakanda Terminology

  • Solution
    • Top level Wakanda item
    • Can hold multiple projects
    • Contains the directory which holds users and groups

  • Project
    • Design-time term/concept representing one application
    • Contains one Datastore Model
    • Project's folder holds multiple files

  • Application
    • Runtime representation of a Wakanda project

Wakanda Terminology

  • Datastore Model
    • Project specific definition of data
    • Describes dataclasses of the datastore, their attributes
      and other related items

  • Datastore
    • Run-time term for an application's server-side bundle of
      data and related code

  • Datastore Class
    • Defines one type of data in the datastore model
    • Composed of datastore class attributes, methods and events
    • Often referred to as just dataclass

Wakanda Terminology

  • Datastore Class Attribute
    • Smallest portion of data in a dataclass
    • Often referred to as just attribute

  • Datastore Class Method
    • A portion of JavaScript code associated with a dataclass
    • Often referred to as a class method

  • Datastore Entity
    • Similar to a record, one instance of information in a dataclass
    • When used in code, a JavaScript object
    • Commonly referred to as an entity

Wakanda Terminology

  • Entity Collection
    • A bundle of from 0 to N entities
    • When used in code, similar to a JavaScript array
    • May be ordered or un-ordered
      • In an un-ordered collection each entity can exist only once
      • In an ordered collection each entity may appear more than once

Wakanda Concept

  • Server-side vs. Browser-side
    • Code in the model runs on Wakanda Server
    • Code incorporated into a web page runs on the browser

    • Unless otherwise noted, all code shown in Day 1 of
      Wakanda Training runs under Wakanda Server
