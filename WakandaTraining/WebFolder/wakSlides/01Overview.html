Day 1 – Intro to Wakanda Server-Side


Wakanda Overview

  • Wakanda is a development and deployment
    environment for data-driven, browser-based
    applications

Wakanda has three parts

  • Wakanda Server
  • Wakanda Ajax Framework (WAF)
  • Wakanda Studio

Wakanda Server

  • Wakanda Server is a faceless Data, Web, and REST server
    with a fully threaded JavaScript environment.

Wakanda Server as a Data Server

  • Built into Wakanda Server
  • NoSQL but still relational
  • Entity/Object based - Data is returned as JavaScript objects

Wakanda Server as a Web Server

  • Built-in admin pages
  • Smart web page distribution (desktop, tablet, smartphone)
  • Automatic session management

Wakanda Server as a REST Server

  • Industry standard HTTP-based data transfer architecture
  • Used by Wakanda Ajax Framework to get and set data in Wakanda Server
  • Can be used by other browser frameworks to manage data

Wakanda Server includes:

  • A fully threaded, very fast javascript environment
  • A complete scriptable backup system
  • A host of server-side services and resources

Wakanda Server runs on

  • Windows
  • Mac
  • Linux

Wakanda Ajax Framework

  • Multiple layers of access to Wakanda Server
  • Provides easy-to-use data retrieval and update
  • Implements a host of Wakanda specific widgets that are data aware
  • Can be used in conjunction with other frameworks

WAF Requirements

  • HTML 5
  • CSS 3
  • Javascript

  • No browser plug-in needed!

WAF Supported Browsers

  • IE 9
  • Chrome 6
  • Safari 5
  • Firefox 3.6

Wakanda Studio

  • Wakanda Studio is an IDE for creating
    Wakanda applications

  • Wakanda Studio runs on Mac OSX 10.6 and
    Windows Vista, 7 or 8 (32 or 64 bit)

Wakanda Studio Editors/Designers

  • Solution Explorer
  • Datastore Model Designer
  • GUI Designer (web page designer)
  • Code editor
  • Users and Groups editor (the Directory)
  • Solution and project settings editor
